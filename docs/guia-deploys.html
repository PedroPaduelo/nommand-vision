Deploys & CI/CD

Gestão de pipelines de build/deploy de projetos com logs em tempo real, variáveis de ambiente secretas, cancelamento e retry.

O que é

O módulo de Deploys gerencia deploys de código via pipeline simples no backend, com estados, logs e ambiente.

Objetivos

  • Disparar deploys por projeto.
  • Salvar estado e métricas de cada deploy.
  • Expor logs em lista e streaming SSE.
  • Operar env vars por deploy com secrets.
  • Suportar cancelamento e retry.

Esquema de banco de dados

As tabelas principais e enums estão em backend/src/db/schema/deploys.ts.

  • deploys: id, projectId, workspaceId, environment (preview/staging/production), status (queued/building/deploying/success/failed/cancelled), branch, commitSha, commitMessage, logs, url, error, durationMs, startedAt, completedAt, triggeredBy, createdAt.
  • deploy_logs: id, deployId, level (info/warn/error), message, timestamp.
  • deploy_env_vars: id, deployId, key, valueEncrypted, isSecret, createdAt.

Rotas do módulo

As rotas estão definidas em backend/src/routes/deploys.ts e usam middleware de autenticação.

  • GET /api/deploys - listar deploys (filtros: projectId, status, environment, page, limit).
  • GET /api/deploys/:id - obter deploy específico.
  • POST /api/deploys - criar deploy (body: projectId, environment?, commitSha?, commitMessage?, branch?).
  • POST /api/deploys/:id/cancel - cancelar deploy (status queued/building/deploying).
  • POST /api/deploys/:id/retry - criar um novo deploy com o mesmo commit/branch.
  • GET /api/deploys/:id/logs - obter últimos 100 logs do deploy.
  • GET /api/deploys/:id/logs/stream - SSE de logs (stream de eventos em tempo real).
  • GET /api/deploys/:id/env - listar env vars do deploy.
  • POST /api/deploys/:id/env - criar/atualizar env var (key,value,isSecret?).
  • DELETE /api/deploys/:id/env/:varId - excluir env var.

Implementação backend

Service principal: backend/src/services/deploys.service.ts. Validações de entrada em backend/src/schemas/deploys.schema.ts.

Exemplos de lógica:

  • create: Valida projectId, insere deploy com status queued e triggeredBy.
  • cancel: só permite queued/building/deploying e seta status cancelled, completedAt.
  • retry: clona deploy antigo com status queued e commit/branch originais.
  • getLogs: lê deploy_logs ordenados por timestamp (limit 100).
  • setEnvVar: upsert em deploy_env_vars, guarda valueEncrypted e isSecret true por padrão.

Logs em tempo real (SSE)

Endpoint GET /api/deploys/:id/logs/stream retorna Content-Type: text/event-stream e envia eventos do tipo data: ...\n\n.

No servidor atual, o stream é simulado (conecta, builds steps, finaliza) e pode ser estendido para injetar logs do worker.

Segurança e restrições

  • Middleware authMiddleware exige usuário autenticado e workspace.
  • Listagem filtra por workspaceId para evitar vazamento entre workspaces.
  • Env vars secretas são marcadas com isSecret e retornam *** no listing.
  • Dados sensíveis em logs devem ser sanitizados antes de gravar ou stream.

Pontos de risco

ALTO

Exposição de variáveis de ambiente

Env vars devem ser criptografadas e não exibidas em texto puro.

MÉDIO

Cancelamento em execução

Cancelamento deve interromper worker e atualizar status atômico. Hoje só atualiza status.

MÉDIO

Builds caros

Sem limites de concorrência pode gerar custos altos. Adicionar fila e throttle.

Como usar (exemplos cURL)

  1. Trigger deploy: 
    curl -X POST /api/deploys -H 'Authorization: Bearer TOKEN' -H 'Content-Type: application/json' -d '{"projectId":"UUID","environment":"staging","branch":"main","commitSha":"abc123"}'
  2. Obter deploy: 
    curl -X GET /api/deploys/:id -H 'Authorization: Bearer TOKEN'
  3. Stream logs: 
    curl -N /api/deploys/:id/logs/stream -H 'Authorization: Bearer TOKEN'
  4. Set env var: 
    curl -X POST /api/deploys/:id/env -H 'Authorization: Bearer TOKEN' -H 'Content-Type: application/json' -d '{"key":"API_KEY","value":"secret","isSecret":true}'
  5. Retry/cancel: 
    curl -X POST /api/deploys/:id/retry -H 'Authorization: Bearer TOKEN'
curl -X POST /api/deploys/:id/cancel -H 'Authorization: Bearer TOKEN'
