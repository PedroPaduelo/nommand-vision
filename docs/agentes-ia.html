Sistema de Agentes IA

Criação, execução e orquestração de agentes

O que são Agentes

Agentes são entidades de IA configuráveis que podem ser ativados perante comandos ou em automações. Cada agente tem seu próprio prompt, modelo, temperature, tools e categoria.

Tabela `agents`

Campo Tipo Descrição
id uuid PK
workspaceId uuid Isolamento por workspace
name varchar(100) Nome do agente
type enum assistant, custom, tool, supervisor
model varchar(50) claude-sonnet-4-20250514
systemPrompt text Instruções personalizadas
config jsonb Configurações específicas
temperature varchar(5) 0.0 a 2.0
maxTokens integer Máx. tokens de resposta
tools jsonb (string[]) Lista de tools disponíveis
status enum active, inactive, error
category enum dev, review, qa, deploy, custom
statsTotalRuns integer Total de execuções
statsSuccessRate varchar(6) Porcentagem de sucesso

Tabela `agent_runs`

Cada execução de agente é registrada com input/output, duração, status e métricas.

Campo Tipo Descrição
id uuid PK
agentId uuid FK para agents
userId uuid Quem executou
input text Prompt/entrada
output text Resposta gerada
tokensUsed integer Total de tokens consumidos
durationMs integer Tempo de execução
status enum running, completed, error, timeout
error text Erro se houver falha

Tipos de Agentes

assistant

Agente genérico de chat. Usa modelo LLM para responder consultas. Ideal para Q&A, brainstorming.

custom

Agente com prompt customizado. Pode ter persona especializada (ex: code reviewer, architect).

tool

Agente que usa ferramentas externas (file editing, shell commands, web search). Ação orientada.

supervisor

Coordena múltiplos agentes. Delega tarefas e agrega resultados (swarm pattern).

Executar um Agente 

POST /api/agents/:id/run
Authorization: Bearer <jwt>
Content-Type: application/json

{
  "input": "Revisar este código...",
  "projectId": "uuid-do-projeto"  // opcional
}

// Resposta:

  "output": "Código revisado...",
  "tokensUsed": 1234,
  "durationMs": 2456,
  "startedAt": "...",
  "completedAt": "..."
}

// Se streaming (nova rota):
POST /api/agents/:id/run/stream
Retorna: text/event-stream com chunks

Validação de Prompt

Antes de executar, o input passa por um sandbox de validação para prevenir injection attacks. 

// prompt-sandbox.service.ts
function validateAndSanitize(input: string, agentType: string) {
  // Regras:
  // - Remove tags HTML perigosas
  // - Detecta prompts de jailbreak
  // - Limita comprimento máximo
  // - Bloqueia comandos do sistema

  if (input.includes('ignore previous instructions')) {
    return { isValid: false, errors: ['Possível jailbreak detectado'] };
  }

  return { isValid: true, sanitized: sanitizeHtml(input) };
}
