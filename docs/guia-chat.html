Chat & Conversations

Sistema de conversas com agentes via sessões, streaming de respostas (SSE), armazenamento de histórico e integração com agent runs.

📋 Visão Geral

Propósito

Permitir que usuários interajam com agentes de IA através de sessões de chat persistidas, com streaming em tempo real das respostas.

Arquitetura

Hono + Drizzle ORM + PostgreSQL. Streaming SSE via Response body com ReadableStream.

Estado

Prototype Implementação mockada - integração com Anthropic API pendente.

Tech Stack

TypeScript, Zod validation, SSE streaming, Hono framework

🗃️ Banco de Dados

Schema Chat (db/schema/chat.ts)

Schemas atualmente utilizados pelo chat.service.ts:

chatSessions

export const chatSessions = pgTable('chat_sessions', {
  id: uuid('id').primaryKey().defaultRandom(),
  workspaceId: uuid('workspace_id').notNull().references(() => workspaces.id),
  projectId: uuid('project_id').references(() => projects.id),
  userId: uuid('user_id').notNull().references(() => users.id),
  title: varchar('title', { length: 200 }).notNull(),
  model: varchar('model', { length: 50 }).notNull().default('gpt-4-turbo'),
  createdAt: timestamp('created_at').notNull().defaultNow(),
  updatedAt: timestamp('updated_at').notNull().defaultNow(),
})

chatMessages

export const chatMessages = pgTable('chat_messages', {
  id: uuid('id').primaryKey().defaultRandom(),
  sessionId: uuid('session_id').notNull().references(() => chatSessions.id),
  role: chatMessageRoleEnum('role').notNull(), // 'user' | 'assistant' | 'system'
  content: text('content').notNull(),
  tokensIn: integer('tokens_in'),
  tokensOut: integer('tokens_out'),
  durationMs: integer('duration_ms'),
  createdAt: timestamp('created_at').notNull().defaultNow(),
})

🔗 Rotas API

Todas as rotas requerem autenticação via authMiddleware.

GET
/api/chat/sessions

Lista sessões do usuário no workspace, com paginação e filtro opcional por projectId.

GET
/api/chat/sessions/:id

Obtém sessão específica com todas as suas mensagens.

POST
/api/chat/sessions

Cria nova sessão de chat. Parâmetros opcionais: title, model, projectId.

DELETE
/api/chat/sessions/:id

Deleta sessão (cascade nas mensagens).

POST
/api/chat/sessions/:id/messages

Envia mensagem e recebe resposta completa (non-streaming). Salva user msg + assistant msg no DB.

POST
/api/chat/sessions/:id/messages/stream

Envia mensagem e recebe resposta via streaming SSE. Content-Type: text/event-stream.

⚙️ Chat Service

Arquivo: backend/src/services/chat.service.ts

listSessions(workspaceId, input)

Lista sessões com query params opcionais: projectId, page, perPage. Retorna { data, meta }.

getSession(workspaceId, id)

Busca sessão por ID + suas mensagens ordenadas por createdAt DESC. Throws se não encontrada.

createSession(workspaceId, userId, input)

Cria nova sessão. Default title: "Nova conversa", model: claude-sonnet-4-20250514.

deleteSession(workspaceId, id)

Deleta sessão. Cascade das mensagens via FK. Retorna { success: true }.

sendMessage(workspaceId, userId, sessionId, input)

Salva mensagem do usuário, gera resposta mockada, salva resposta do assistente. Calcula tokens (length/4) e duração. Atualiza updatedAt da sessão.

streamMessage(workspaceId, userId, sessionId, input)

Chama sendMessage e retorna ReadableStream com SSE. Divide resposta em palavras, delay 30ms entre chunks.

⚠️ Implementação Mockada

O generateMockResponse simula respostas baseadas em palavras-chave. NÃO há integração real com Claude API ou agents.service.run().

🔒 Segurança

Controle de Acesso

  • workspaceId filter: Middleware authMiddleware injeta user.workspaceId em todas as queries.
  • userId filter: Sessões filtradas por userId implicitamente via query implícita (NOTA: chatSessions NÃO tem filtro por userId no serviço!).
  • Row isolation: Deleção em cascade preserva integridade referencial (ON DELETE CASCADE).

Riscos Identificados

🔴 RIDÍCULO: Vazamento de Mensagens

listSessions e getSession usam apenas eq(chatSessions.workspaceId). NÃO filtram por userId! Usuários podem ver sessões de outros no mesmo workspace.

Mitigação URGENTE: Adicionar eq(chatSessions.userId, userId) em todas as queries.
🟡 Streaming Token Leak

Streaming SSE não tem timeout configurado. Conexões órfãs podem consumir recursos.

Mitigação: Implementar timeout no controller + cleanup de streams.
🟡 Model Injection

Parâmetro model é opcional e não validado contra lista permitida. Usuário poderia solicitar modelo inexistente ou caro.

Mitigação: Whitelist de modelos válidos por workspace/project.
🟢 Storage de Dados Sensíveis

Mensagens armazenadas em texto plano. Se houver dados sensíveis, não há criptografia em repouso.

Mitigação: Considerar encrypt-at-rest para dados regulados.

📖 Como Usar

Para Desenvolvedores Frontend

  1. Criar sessão: POST /api/chat/sessions com {"title": "Meu Chat", "model": "claude-sonnet-4-20250514"}
  2. Enviar mensagem via stream: 
    const response = await fetch('/api/chat/sessions/:id/messages/stream', {
  method: 'POST',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify({content: message})
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const {done, value} = await reader.read();
  if (done) break;
  const text = decoder.decode(value);
  // Parse SSE: "data: {...}\n\n"
  const lines = text.split('\n\n');
  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const data = line.slice(6);
      if (data === '[DONE]') return;
      console.log(JSON.parse(data).content);
    }
  }
}
  3. Listar sessões: GET /api/chat/sessions?projectId=:id&page=1&perPage=20
  4. Deletar: DELETE /api/chat/sessions/:id
