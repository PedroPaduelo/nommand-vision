🤖 Agents & Execuções

Sistema completo para criação, configuração e execução de agentes de IA. Suporta múltiplos tipos de agentes, personalização de prompts, ferramentas externas e tracking detalhado de execuções.

🎯

Propósito

Permitir que usuários criem e configurem agentes de IA personalizados com diferentes tipos, modelos, prompts system e ferramentas.

⚙️

Configuração

Definir tipo de agente, modelo Claude, temperatura, maxTokens, tools array e systemPrompt para customizar o comportamento.

▶️

Execução

Executar agentes via API com entrada de texto, opcionalmente vinculado a projetos ou usuários específicos.

📊

Tracking

Registrar cada execução (AgentRun) com duração, tokens usados, status e output para análise posterior.

📈

Estatísticas

Acumular métricas agregadas: total de runs, duração média e taxa de sucesso por agente.

🔔

Notificações

Criar notificações automaticamente após conclusão de execuções importantes ou erros.

🔒 Segurança

🏢

Isolamento por Workspace

Todas as queries filtram por workspaceId. Usuários só acessam agentes do próprio workspace via middleware.

// agentsRouter - proteção em todas as rotas
agentsRouter.use('/*', authMiddleware)
// authMiddleware garante: user.workspaceId disponível
✅

Validação de Estado

Execução verifica se agente está ativo:

if (!agent.isActive) {
  throw new Error('Agent is not active')
}
🔑

Autenticação JWT

Todas as rotas requerem token válido. Tokens contêm userId e workspaceId válidos.

🚫

Rate Limiting

Aplicado globalmente via middleware (detalhes implementação). Recomenda-se lim por workspace para agents.

📝

Audit de Criação

Campo createdBy rastreia autor de cada agente. createdAt e updatedAt automáticos.

🔒

Cascade Delete

Agent removido → agentRuns deletados automaticamente via onDelete: 'cascade' no schema.

⚠️ Riscos Identificados

ALTO

Prompt Injection

Usuários controlam systemPrompt em agentes custom. Risco: vazar instruções internas ou bypass de segurança.

Mitigação: Sandbox de prompts, scan por padrões suspeitos, warning no UI, limitar customização em workspaces com permissão
ALTO

Custo API Anthropic

Sem limite de tokens input/output. Ataque DoS com prompts enormes gera custo astronômico.

Mitigação: Impor maxTokens no input + output, rate limit por user/projeto, quota diária por workspace, cache de respostas similares
MÉDIO

Anthropic SDK Não Implementado

Código atual simula resposta. Chamada real a Anthropic API ainda não integrada.

Mitigação: Implementar Anthropic SDK com erro handling, timeout, retry logic, api key isolation por workspace
MÉDIO

Tools Array sem Validação

tools: z.array(z.string()) não valida se tool realmente existe/autorizada.

Mitigação: Whitelist de tools predefinidas, verificar contra registry antes de execução
MÉDIO

Leak de Dados via Output

Agent output pode conter dados sensíveis do contexto (outros messages, workspace info).

Mitigação: Sanitização de output, redaction de padrões sensíveis, log audits
BAIXO

SQL Injection Indirecta

Drizzle ORM com prepared Statements protege, mas queries dinâmicas em search precisam de cuidado.

Mitigação: LIKE parametrizado já implementado, validar search length
BAIXO

Race Condition em Update Stats

Múltiplos runs simultâneos podem recalcular stats com dados desatualizados.

Mitigação: Usar UPDATE com cálculos SQL nativos ou locks ou buffer de stats

📈 Monitoramento & Métricas

Estatísticas por Agente (acumuladas)

  • statsTotalRuns: Total de execuções concluídas (incrementado em cada run)
  • statsAvgDurationMs: Média móvel de duração (recalculada via updateStats)
  • statsSuccessRate: Porcentagem de sucesso (0-100, atualizada em cada run)

Função updateStats (agents.service.ts)

async function updateStats(id: string, stats: { duration: number; success: boolean }) {
  const agent = await getRaw(id)

  const totalRuns = (Number(agent.statsTotalRuns) || 0) + 1
  const currentAvg = Number(agent.statsAvgDurationMs) || 0
  const newAvg = Math.round((currentAvg * (totalRuns - 1) + stats.duration) / totalRuns)

  const currentSuccessRate = Number(agent.statsSuccessRate) || 100
  const successCount = Math.round((currentSuccessRate * (totalRuns - 1)) / 100) + (stats.success ? 1 : 0)
  const newSuccessRate = (successCount / totalRuns) * 100

  await db.update(agents)
    .set({
      statsTotalRuns: totalRuns,
      statsAvgDurationMs: newAvg,
      statsSuccessRate: newSuccessRate.toFixed(2),
      updatedAt: new Date(),
    })
    .where(eq(agents.id, id))
}

Tipos de Agente

  • assistant — Agente geral, útil para tarefas variadas
  • custom — Prompt personalizado, máxima flexibilidade
  • tool — Focado em uma ferramenta específica (ex: search)
  • supervisor — Coordena múltiplos agentes (orquestrador)
