Fluxo de Autenticação

JWT, Sessions, Rate Limiting e Segurança

Sistema de Autenticação

O sistema usa JWT (JSON Web Tokens) para autenticação stateless com suporte a refresh tokens. Tokens de acesso duram 15 minutos, refresh tokens duram 7 dias.

Access Token
15 min
Refresh Token
7 dias

Estrutura do JWT 

// Access Token Payload
{
  "userId": "uuid-do-usuario",
  "workspaceId": "uuid-do-workspace",
  "email": "user@email.com",
  "role": "Frontend",
  "iat": 1700000000,
  "exp": 1700000900  // +15 min
}

// Refresh Token Payload
{
  "userId": "uuid-do-usuario",
  "type": "refresh",
  "sessionId": "uuid-da-sessao",
  "iat": 1700000000,
  "exp": 1700600000  // +7 dias
}

Middleware de Autenticação 

// Fluxo do Auth Middleware
1. Extrai token do header "Authorization: Bearer <token>"
2. Verifica se token existe → 401 se não existir
3. Valida JWT com verifyAccessToken()
4. Se inválido/expirado → 401
5. Verifica blacklist de tokens (logout)
6. Se válido → adiciona user ao contexto
7. Permite requisição prosseguir

// Proteção por Workspace
- Todas rotas usam authMiddleware
- Queries são filtradas por workspaceId do usuário
- Isolamento de dados entre workspaces
