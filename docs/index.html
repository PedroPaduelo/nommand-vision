Pontos de Risco & Mitigações

Análise de riscos identificados no sistema e recomendações de mitigação.

ALTO

Rate Limit Insuficiente

Auth endpoints têm rate limit por IP, mas workspaces compartilham limites. Possível DoS por workspace.

Mitigação: Rate limit por usuário + workspace separadamente
ALTO

JWT sem Revogação Imediata

Refresh tokens estão em BD mas access tokens não têm blacklist. Logout não invalida access token imediatamente.

Mitigação: Token blacklist com TTL ou expiry muito curto (5min)
MÉDIO

SQL Injection via JSON

Drizzle ORM protege queries, mas queries dinâmicas com JSONB podem ser vulneráveis se não validadas.

Mitigação: Zod validation estrito + Drizzle parameterized queries
MÉDIO

Leak de Dados Cross-Workspace

Middleware auth verifica workspaceId, mas queries podem falhar em edge cases. Isolamento pode falhar.

Mitigação: Audit sistemático de todas as queries + Row Level Security (RLS)
MÉDIO

Agent Prompt Injection

Usuário controla systemPrompt em agentes customizados. Pode instructions de sistema e causar behavior indesejado.

Mitigação: Sandbox de prompts + review humano + allowed list de instruções
MÉDIO

Custos de IA Descontrolados

Sem limites de tokens ou custo, agentes podem gerar custos excessivos em Anthropic API.

Mitigação: Budget limits por workspace + alertas em tempo real
BAIXO

Logs com Dados Sensíveis

Logs podem conter tokens, PII ou dados sensíveis se não sanitizados.

Mitigação: Auto-sanitization no logger + scrubber de padrões sensíveis
BAIXO

Deploy Falhos Silenciosos

Deploys podem falhar sem notificação adequada ao usuário.

Mitigação: Notificações obrigatórias + status dashboard
BAIXO

Webhooks sem Retry

Webhooks externos podem falhar sem retry automático, causando perda de eventos.

Mitigação: Implementar dead letter queue + retry exponencial
