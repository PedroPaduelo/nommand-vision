Pipeline de Deploy

Fluxo completo de build, deploy e streaming de logs

Sistema de Deploy

O sistema suporta ambientes: Preview, Staging e Production. Cada deploy gera logs em tempo real via WebSocket e pode ser acompanhado via UI.

Preview Staging Production

Ciclo de Vida do Deploy

Trigger (Manual/Branch Push)
↓
Status: queued
↓
Build (builder)
↓
Status: building
↓
Deploy para ambiente
↓
Status: deploying
↓
Health check
↓
Status: success ou failed

Tabela `deploys`

Campo Tipo Descrição
id uuid PK única
projectId uuid FK para projetos (cascade)
workspaceId uuid FK para workspaces
environment enum preview, staging, production
status enum queued, building, deploying, success, failed, cancelled
branch varchar(100) Branch do git
commitSha varchar(40) Hash do commit
commitMessage text Mensagem do commit
logs text Logs completos (pode ser grandes)
url varchar(500) URL do deploy (ex: https://project--branch.ddw1sl.easypanel.host)
error text Erro em caso de falha
durationMs integer Duração em ms
startedAt timestamp Início do deploy
completedAt timestamp Fim do deploy
triggeredBy uuid → users Quem iniciou

Tabela `deploy_logs`

Logs associados a cada deploy, com nível e timestamp.

Campo Tipo Descrição
id uuid PK
deployId uuid FK para deploys (cascade)
level enum info, warn, error
message text Mensagem do log
timestamp timestamp Quando o log foi gerado

Tabela `deploy_env_vars` (Secrets)

Variáveis de ambiente específicas por deploy. Valores são criptografados (crypto-js ou similar).

Campo Tipo Descrição
id uuid PK
deployId uuid FK para deploys
key varchar(100) Nome da variável (ex: DATABASE_URL)
valueEncrypted text Valor criptografado
isSecret boolean Se deve ser tratada como secret

Streaming de Logs via WebSocket

Durante o build/deploy, logs são enviados em tempo real via WebSocket para o frontend. 

// Canais WebSocket usados:
- deploy:${deployId}   → Logs daquele deploy
- deploy:progress      → Progresso geral
- deploy:status        → Atualizações de status

// Mensagem exemplo:
{
  "event": "deploy:log",
  "data": {
    "deployId": "uuid",
    "level": "info",
    "message": "Building Docker image...",
    "timestamp": "2025-03-20T..."
  }
}

// Frontend se inscreve em:
const ws = new WebSocket(`/ws?token=${jwt}`);
ws.on('message', (msg) => {
  const { event, data } = JSON.parse(msg);
  if (event === 'deploy:log') {
    appendLog(data.message);
  }
});

Iniciar Deploy via API 

POST /api/deploys
Authorization: Bearer <jwt>
Content-Type: application/json

{
  "projectId": "uuid-do-projeto",
  "environment": "staging" | "production",
  "branch": "main",
  "commitSha": "abc123...",
  "commitMessage": "Fix bug in login"
}

// Resposta:
{
  "id": "uuid-do-deploy",
  "status": "queued",
  "environment": "staging",
  "createdAt": "..."
}
