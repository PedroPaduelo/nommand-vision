📊 Analytics & Metrics

Módulo de coleta e visualização de métricas de uso: execuções de agentes, tokens consumidos, custos, deploys (sucesso/falha), atividade ao longo do tempo (time series), top projects e top agents.

🗄️ Schema do Banco de Dados

analyticsEvents

Tabela de eventos brutos para rastreamento de todas as ações no sistema.

Campo Tipo Obrigatório Descrição
id uuid Sim PK auto-generated
workspaceId uuid Sim FK para workspaces - isolamento por workspace
projectId uuid Não FK para projects - evento pode estar associado a um projeto
agentId uuid Não ID do agente relacionado ao evento
eventType varchar(50) Sim Tipo do evento: deploy, message, agent_action, token_usage
eventData jsonb Não Dados estruturados do evento (status, tokens, agent_id, agent_name, etc)
userId uuid Não FK para users - usuário que realizou a ação
createdAt timestamp Sim Data de criação do evento

dailyMetrics

Métricas agregadas diariamente - preenchido por job noturno para otimizar queries.

Campo Tipo Obrigatório Descrição
id uuid Sim PK auto-generated
workspaceId uuid Sim FK para workspaces
date date Sim Data da métrica (YYYY-MM-DD)
deploysCount integer Sim Total de deploys no dia
deploysSuccess integer Sim Deploys com status success
deploysFailed integer Sim Deploys com status failed
agentsRuns integer Sim Total de execuções de agentes
tokensUsed integer Sim Total de tokens consumidos
avgResponseTimeMs integer Sim Tempo médio de resposta em milissegundos
activeUsers integer Sim Quantidade de usuários ativos no dia

analyticsSnapshots

Snapshots de métricas para períodos agregados (daily, weekly, monthly).

Campo Tipo Obrigatório Descrição
id uuid Sim PK auto-generated
workspaceId uuid Sim FK para workspaces
period enum Sim Período: daily, weekly, monthly
metrics jsonb Sim Objeto com: total_deploys, success_rate, active_agents, total_messages, avg_response_time, uptime
snapshotDate date Sim Data do snapshot

🔒 Segurança e Isolamento

  • Workspace Filter: Todas as queries filtram por workspaceId do usuário autenticado
  • Auth Middleware: Todas as rotas requerem autenticação JWT válida
  • Row-level Security: FK constraints com ON DELETE CASCADE para isolamento
  • Input Validation: Zod schemas validam todos os inputs das requisições

⚠️ Riscos e Mitigações

ALTO

PII em eventData

Dados sensíveis podem ser armazenados no campo jsonb eventData.

Mitigação: Sanitizar dados antes de salvar, implementar política de retenção, criptografia em repouso.
ALTO

Volume de Eventos Crescente

Alto volume de eventos pode impactar performance e custo de armazenamento.

Mitigação: Implementar TTL/política de retenção, agregar para dailyMetrics, usar particionamento.
MÉDIO

Precisão de Agregação

Job noturno pode perder eventos de último momento do dia.

Mitigação: Janela de agregação com overlap, recompute de métricas ao vivo quando necessário.
MÉDIO

Performance de Queries Timeseries

Queries em grandes intervalos podem ser lentas em analyticsEvents.

Mitigação: Usar dailyMetrics para intervals grandes, índices em (workspaceId, eventType, createdAt).
BAIXO

Custo de Armazenamento

Jsonb consome mais storage que colunas tipadas.

Mitigação: Migrar campos conhecidos para colunas, compressão, archiving de dados antigos.

📖 Como Usar

1. Consultar Dashboard

GET /api/analytics/dashboard?period=30d

2. Obter Dados para Gráficos

GET /api/analytics/timeseries?metric=deploys&period=7d
GET /api/analytics/timeseries?metric=messages&period=30d
GET /api/analytics/timeseries?metric=errors&period=90d

3. Consultar Custos

GET /api/analytics/tokens

4. Rastrear Eventos Personalizados

POST /api/analytics/track
{
  "eventType": "button_click",
  "projectId": "uuid-do-projeto",
  "eventData": {
    "button_id": "deploy-now",
    "page": "dashboard"
  }
}

5. Integração com Frontend (Agents Service)

O agents.service deve chamar analytics.trackEvent após cada execução:

import { trackEvent } from './analytics.service.js'

// Após execução de agente
await trackEvent(workspaceId, userId, {
  eventType: 'agent_action',
  eventData: {
    agent_id: agent.id,
    agent_name: agent.name,
    action: 'run',
    input_tokens: usage.input_tokens,
    output_tokens: usage.output_tokens
  }
})
