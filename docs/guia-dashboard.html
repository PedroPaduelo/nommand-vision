Guia: Panorama (Dashboard)

Visão estruturada do módulo Panorama/Dashboard do sistema Nommand Vision.

O que é

O Panorama é a página inicial do sistema, serving as dashboard principal após o login. Exibe KPIs, métricas de performance, status de deploys, atividade recente e recursos computacionais do workspace.

Estrutura da Página

A página está organizada em pages/Panorama/ (não utiliza o padrão features/dashboard). A estrutura é:

sistema-react/src/pages/Panorama/
├── index.tsx           # Componente principal
└── Panorama.module.css # Estilos CSS Modules

Layout - Grid de Widgets

O Panorama utiliza um layout de grid responsivo com 3 linhas principais:

  • Row 4 (4 colunas): 4 cards de métricas (Status, Requests 24h, Latência, Uptime)
  • Row 2 (2 colunas): Stack ativo + Compute Power
  • Row 3 (3 colunas): Atividade recente + Deploys + Terminal

Componentes

A página Panorama usa componentes locais e globais:

Componente Tipo Descrição
Sparkline Local Gráfico de迷你 linha para tendências (requests, latency)
UptimeBar Local Barra de uptime com 30 dias de status
Tour Local Overlay de onboarding interativo (4 passos)
AppShell Global (template) Layout com Sidebar, Header, CommandPalette, NotificationPanel

Dados Consumidos

A página consome dados via React Query hooks:

Hook Origem Dados Retornados
useMetrics('30d') useAnalyticsQueries.ts totalRequests, avgLatency, uptime (DashboardData)
useAuth() useAuth.ts user (userName, cpuLevel, stack, tourDone)

Fluxo de Carregamento

  1. Usuário faz login (AuthGuard verifica user.authenticated)
  2. Sistema verifica onboarding (OnboardingGuard verifica user.onboarded)
  3. Se não fez tour, exibe Tour overlay com 4 passos
  4. Carrega métricas via useMetrics('30d') - 30s staleTime
  5. Exibe dados do usuário (nome, stack, CPU)
  6. Atualiza latência a cada 3s (setInterval)

Integrações API

O Panorama faz chamadas para os seguintes endpoints:

Método Endpoint Descrição
GET /api/analytics/dashboard?period=30d Métricas do dashboard (requests, latency, uptime, deploys, agents)

Estado Global

O sistema utiliza:

  • React Query (@tanstack/react-query) - para caching e sincronização de dados
  • Zustand/Context - para estado global (AuthContext, ThemeContext, I18nContext)
  • staleTime: 30 segundos para todas as queries

Segurança

A rota /panorama é protegida por dois guards:

  • AuthGuard - verifica se user.authenticated === true, caso contrário redireciona para /login
  • OnboardingGuard - verifica se user.onboarded === true, caso contrário redireciona para /onboarding

Dark Mode

O dark mode é obrigatório e já está implementado via CSS custom properties (variáveis como --neon, --green, --panel, --border, --muted). O toggle já existe no Header.

Fluxo do Usuário

  1. Usuário acessa a aplicação
  2. Se não autenticado → redirecionado para /login
  3. Se autenticado mas não onboarded → redirecionado para /onboarding
  4. Após onboarding completo → redirecionado para /panorama (rota padrão/index)
  5. No panorama, pode navegar para outros módulos via Sidebar

Riscos e Considerações

  • Excesso de requests: A página faz 1 query principal (analytics/dashboard). Para evitar sobrecarga, o staleTime é de 30s.
  • Falta de caching: React Query gerencia cache automaticamente com staleTime.
  • Dados demo: Alguns componentes (Sparkline, UptimeBar) usam dados simulados (Math.random) apenas para efeito visual - não são dados reais da API.
  • Latência em tempo real: O valor de latência é simulado (atualiza a cada 3s), não é uma conexão real de WebSocket.
