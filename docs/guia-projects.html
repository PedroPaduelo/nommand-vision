PROJECTS & GIT

Gerenciamento completo de projetos de código, integração com Git, definição de stack tecnológica e associação de agentes especializados.

🎯

Propósito

O módulo Projects centraliza o gerenciamento de projetos de código, permitindo definir frameworks, stacks tecnológicas, associar agentes especializados e rastrear deploys e execuções.

🔐

Segurança

Isolamento por workspaceId garante que usuários só acessem projetos do próprio workspace. Filtros obrigatórios em todas as queries.

🔄

Integrações

Integração nativa com Deploys (projectId) e AgentRuns (projectId) para rastreabilidade completa do ciclo de vida.

⚠️

Riscos

Soft delete via deletedAt, conflitos de slug único e validação de permissões por workspace são pontos de atenção.

📖 Como Usar

Fluxo de Criação de Projeto

  1. Acesse a dashboard de projetos do seu workspace
  2. Clique em "Novo Projeto"
  3. Preencha: Nome (obrigatório), Descrição, Framework (React, Node, Django...)
  4. Defina a Stack Tecnológica (array de strings)
  5. Associe Agentes (selecionando da lista disponível)
  6. Escolha o Role (Frontend, Backend, Design, Data)
  7. O Slug é gerado automaticamente (minha-app-123)
  8. Clique "Criar" → projeto criado com status 'active'

Associação de Agentes

O campo agentIds armazena um array de UUIDs de agentes vinculados ao projeto:

3Vincular ao projetoPOST /api/projects/ com agentIds: ['uuid1', 'uuid2']
4Alterar associaçõesPUT /api/projects/:id com agentIds atualizado

Visualização de Stats e Activity

  • Stats: GET /api/projects/:id/stats → { commits, deploys, uptime, lastDeploy }
  • Activity: GET /api/projects/:id/activity → timeline de eventos
  • Atualmente ambos retornam dados placeholders (será expandido com integração real de commits/deploys)

🔒 Segurança

Workspace Isolation (Multi-tenancy)

Todas as operações filtram por workspaceId do usuário autenticado (via authMiddleware).

updateProjectWHERE workspaceId = user.workspaceId AND id = ? AND deletedAt IS NULL
deleteProjectWHERE workspaceId = user.workspaceId AND id = ? AND deletedAt IS NULL
getProjectStatsWHERE workspaceId = user.workspaceId AND id = ? AND deletedAt IS NULL
getProjectActivityWHERE workspaceId = user.workspaceId AND id = ? AND deletedAt IS NULL

Outras Validações

  • Slug único: Verificado por generateUniqueSlug (adiciona sufixo numérico se conflito)
  • CreatedBy tracking: Registra userId do criador
  • AgentIds validation: Schema valida que IDs são strings válidas (formato UUID)

⚠️ Pontos de Risco Identificados

MÉDIO

Race Condition em Slug Generation

generateUniqueSlug pode ter race condition em alta concorrência (INSERT antes da verificação completa).

Mitigação: Unique constraint em (workspaceId, slug) no DB + retry logic
MÉDIO

Leak de AgentIds Invalid

agentIds são validated como string[], mas não verifica se UUIDs pertencem ao workspace.

Mitigação: Validar que cada agentId existe no workspace antes de vincular
ALTO

Soft Delete vs Cascades Inconsistent

Deploys tem CASCADE (deleta quando projeto é soft-deleted? NAO - deleta só se hard delete). agentRuns tem SET NULL (OK). Inconsistência pode causar órfãos visíveis.

Mitigação: Documentar: soft delete nao afeta deploys (ainda visíveis). Considerar hide logic no frontend.
BAIXO

Stats Placeholder Inconsistente

getProjectStats retorna dados estáticos; getProjectActivity retorna apenas criação. Nao reflete deploys reais.

Mitigação: Implementar queries reais (deploys count, agentRuns agregados)

💡 Exemplos Práticos

Criar Projeto (cURL)

curl -X POST https://api.nommand.com/api/projects/ \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "E-commerce Frontend",
    "description": "Loja virtual React",
    "framework": "React",
    "stack": ["TypeScript", "TailwindCSS", "React Query"],
    "agentIds": ["a1b2c3d4-e5f6-7890-abcd-ef1234567890"],
    "role": "Frontend",
    "branch": "main"
  }'

Atualizar Stack (cURL)

curl -X PUT https://api.nommand.com/api/projects/{projectId} \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "stack": ["TypeScript", "Vite", "Zustand", "Vitest"],
    "status": "active"
  }'

Listar Projetos (cURL)

curl -X GET "https://api.nommand.com/api/projects/?page=1&perPage=20&status=active" \
  -H "Authorization: Bearer {token}"
