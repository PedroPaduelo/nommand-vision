PLAYGROUND

Ambiente de edição de código integrado com preview em tempo real, terminal emulado e assistente AI. Permite experimentação de stacks web sem sair da plataforma.

🎯

Propósito

O Playground oferece um editor de código integrado com sistema de arquivos virtuais, preview renderizado e terminal simulado, permitindo aos usuários testar e prototipar interfaces e componentes de forma interativa.

📦

Stack

React (TypeScript) + Tailwind CSS. Editor customizado sem Monaco/CodeMirror (linhas simples). Preview via innerHTML sanitizado. Terminal emulado via dicionário de comandos.

🤖

AI Integration

Assistente AI integrado na barra inferior do editor. Permite prompts que adicionam linhas de código via diff visual. Processamento simulado com timeouts.

🔒

Segurança

Sanitização de HTML com DOMPurify antes de renderizar preview. Forbidden tags: script, iframe, form, input. Nenhuma execução real de código ou comandos.

🗃️ Arquitetura do Módulo

O Playground é 100% frontend, rodando inteiramente no navegador. Não há integração com APIs backend para salvar ou carregar arquivos — todo o estado é local (React hooks).

Componente Responsabilidade Props Principais
index.tsx Container principal. Gerencia estado global: arquivos abertos, codeMap, terminal lines, AI processing. Nenhuma (página raiz)
FileTreePanel.tsx Painel esquerdo com árvore de arquivos virtuais. Navegação por tabs nao implementada (abre arquivos diretamente). files (TreeFile[]), activeFile, onFileClick
EditorPanel.tsx Editor de código customizado (sem syntax highlighting). Mostra tabs abertas, linhas numeradas, barra de AI. codeLines, lang, onAiSubmit, aiProcessing
PreviewPanel.tsx Renderiza HTML do preview via dangerouslySetInnerHTML (sanitizado). Mostra badge de Hot Reload. previewHtml (string pré-definida)
TerminalPanel.tsx Terminal emulado com input. Interpreta comandos via dicionário termCommands. lines (TermLine[]), onCommand

🔗 Integrações com Backend

Observação: O Playground atual é isolado do backend. Não há integração real com Projects, Deploys ou API.

Todos os dados são estáticos (hardcoded) no componente index.tsx:

  • treeFiles: estrutura de arquivos simulada
  • initialCodeMap: conteúdo inicial de cada arquivo
  • previewHtml: HTML renderizado no preview (não derivado do código)
  • termCommands: dicionário de comandos e respostas

🔒 Segurança

Sanitização HTML (DOMPurify)

Localizado em src/utils/sanitize.ts. Aplicado tanto no PreviewPanel quanto no TerminalPanel antes de usar dangerouslySetInnerHTML.

ConfigValor
ALLOWED_TAGSp, br, b, i, u, strong, em, code, pre, h1–h6, ul, ol, li, a, span, div, table, thead, tbody, tr, th, td, blockquote, hr, img
ALLOWED_ATTRhref, src, alt, title, class, id, target, rel, width, height
FORBID_TAGSscript, style, iframe, form, input, button, object, embed
FORBID_ATTRonerror, onload, onclick, onmouseover, onfocus, onblur
ALLOW_DATA_ATTRfalse

Riscos Remanescentes

  • CSS Injection: Even com tags sanitizadas, CSS inline dentro de span/div pode alterar layout da página (isolation via shadow DOM não implementado).
  • Eventos via URL: Links com javascript: são removidos, mas href malicioso pode abrir páginas externas.
  • Resource Loading: img src pode apontar para URLs externas (tracking/exfiltration possível via beacon).
  • State isolation: Como não há persistência e o código roda apenas no cliente, risco é limitado ao próprio XSS.

📖 Como Usar

Fluxo do Usuário

  1. Navegar até Playground via menu lateral
  2. Ver o Explorador de Arquivos à esquerda (FileTreePanel)
  3. Clicar em um arquivo para abri-lo no editor (acentivo)
  4. Editar código no painel central (alterações ficam apenas em memória)
  5. Ver o painel Preview à direita (renderização HTML estática)
  6. Abrir o Terminal (aba inferior direita) para comandos simulados
  7. Usar a barra de AI no rodapé do editor para sugestões simuladas

Limitações Atuais

  • Código editado não afeta a preview (previewHtml é constante)
  • Arquivos não são salvos em lugar algum (refresh perde alterações)
  • Editor não possui syntax highlighting
  • AI é apenas mock (adiciona comentário no arquivo ativo)
