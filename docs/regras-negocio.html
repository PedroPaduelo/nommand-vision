Regras de Negócio

Domínios, validações e workflows do sistema

Domínio: Autenticação

Registro de Usuário

  • • Email deve ser único no sistema
  • • Senha mínimo 8 caracteres
  • • Nome 2-100 caracteres
  • • Ao criar, também cria Workspace automaticamente

Login

  • • Aceita email + senha
  • • Retorna JWT (15 min) + Refresh Token (7 dias)
  • • Rate limit: 10 req/min por IP

Sessões

  • • Refresh token é armazenado no DB (sessions table)
  • • Logout remove token da DB e blacklist
  • • Tokens expirados são adicionados à blacklist automaticamente

Domínio: Projetos

Criação

  • • Nome: 2-100 caracteres, único no workspace
  • • Slug: gerado automaticamente (kebab-case)
  • • Status: active ou draft
  • • Campos opcionais: description, icon, iconColor, branch, framework, stack

Relacionamentos

  • • Pertence a um workspace
  • • Criado por um usuário
  • • Pode ter múltiplos deploys, agentes, chats

Domínio: Agentes

Criação

  • • Nome: 2-100 caracteres
  • • Tipo: assistant, custom, tool, supervisor
  • • Categoria: dev, review, qa, deploy, custom
  • • Model padrão: claude-sonnet-4-20250514

Execução

  • • Input mínimo 1 caractere
  • • Rate limit: 30 req/min por workspace
  • • Prompts passam por sanitização (anti-jailbreak)
  • • Tokens estimados e limitados por workspace

Estatísticas

  • • Total de execuções (statsTotalRuns)
  • • Tempo médio (statsAvgDurationMs)
  • • Taxa de sucesso (statsSuccessRate)

Domínio: Deploys

Ambientes

  • • preview: Branchs feature (URL automática)
  • • staging: Homologação
  • • production: Produção

Status

  • • queued: Na fila
  • • building: Build em andamento
  • • deploying: Fazendo deploy
  • • success: Concluído
  • • failed: Falhou
  • • cancelled: Cancelado

Variáveis de Ambiente

  • • Criptografadas no banco (valueEncrypted)
  • • Marcadas como secret (não expostas na UI)
  • • Por deploy (isoladas)

Domínio: Automações

Triggers

  • • schedule: Cron jobs
  • • webhook: HTTP POST
  • • event: Eventos do sistema
  • • manual: Execução manual

Configuração

  • • Actions: array JSON de ações a executar
  • • Pode habilitar/desabilitar (enabled)
  • • Registra runs com status e output

Domínio: Workspaces (Multi-Tenancy)

Estrutura

  • • Cada usuário pertence a um workspace
  • • Workspace pode ter múltiplos usuários
  • • Todos os dados são filtrados por workspaceId
  • • Isolamento total entre workspaces

Planos

  • • free: Limites básicos
  • • pro: Mais recursos
  • • enterprise: Ilimitado
