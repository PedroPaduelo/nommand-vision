Orquestração

Orquestra múltiplos agentes, coordena fluxos complexos e pipelines de trabalho.

📋 Descrição

Orquestra múltiplos agentes, coordena fluxos complexos e pipelines de trabalho.

🎯 Responsabilidades

  • Definir sequência de agentes
  • Passar contexto entre stages
  • Tratar erros e retries
  • Paralelizar quando possível
  • Agregar resultados finais

🔧 Tecnologias

  • BullMQ
  • Temporal
  • Airflow
  • Step Functions
  • Custom DAG

⚡ Como Usar

  • Acesse a página Agentes
  • Selecione Orchestrator
  • Forneça input ou contexto
  • Execute e aguarde resultado

🔄 Fluxo de Execução

  1. Recebe workflow definition: Recebe workflow definition
  2. Cria tarefas para cada agente: Cria tarefas para cada agente
  3. Executa em ordem ou paralelo: Executa em ordem ou paralelo
  4. Passa output como input próximo: Passa output como input próximo
  5. Finaliza com resultado consolidado: Finaliza com resultado consolidado

📡 API Endpoint

POST /api/agents/:id/run

Exemplo de requisição:

{ "input": "workflow: code-review -> test -> deploy", "projectId": "uuid" }

📊 Dados Coletados

  • Entrada: Prompt, arquivos, contexto do projeto
  • Saída: Resultado processado (código, relatório, etc.)
  • Métricas: Total de execuções, taxa de sucesso, tempo médio, tokens usados
  • Logs: Histórico completo de cada execução
