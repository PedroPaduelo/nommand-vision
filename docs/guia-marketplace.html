MARKETPLACE

Catálogo de Agentes, Templates, Integrações e Plugins

Guia: Marketplace

Catálogo centralizado para descoberta e instalação de agentes pré-construídos, templates de projeto, integrações e plugins no workspace.

O que é

O Marketplace é um módulo que permite aos usuários navegar, instalar e gerenciar itens pré-configurados para estender as capacidades do sistema. Cada item pode ter configurações personalizadas que são armazenadas por workspace.

Categorias de Itens

🤖

Agent

Agentes de IA pré-configurados com prompts, tools e modelos definidos. Appecem na lista de agentes disponíveis após instalação.

📁

Template

Templates de projeto com estrutura, configurações e agentes pré-definidos para iniciar novos projetos rapidamente.

🔗

Integration

Integrações com APIs externas (webhooks, serviços terceiros). Ativam capacidades de automação e conexão com ferramentas.

🧩

Plugin

Extensões do sistema que adicionam funcionalidades customizadas. Podem modificar o comportamento de outros módulos.

Como Usar

  1. Acesse a página Marketplace através do menu de navegação
  2. Utilize as categorias (security, quality, productivity, devops, data, docs) para filtrar itens
  3. Use a barra de busca para encontrar itens por nome, descrição ou tags
  4. Visualize detalhes do item: autor, versão, rating, downloads, tags
  5. Clique em "Install" para instalar o item no workspace atual
  6. Itens instalados aparecem com status "Installed" e podem ser utilizados
  7. Gerencie instalações via endpoint /api/marketplace/installed

Endpoints da API

Método Rota Descrição Parâmetros
GET /api/marketplace Lista itens do marketplace category (agent/template/integration/plugin)
search (termo de busca)
sort (name/downloads/rating/created)
page, perPage (paginação)
GET /api/marketplace/installed Lista itens instalados no workspace -
GET /api/marketplace/:id Obtém detalhes de um item específico id (UUID do item)
POST /api/marketplace/:id/install Instala item no workspace id (UUID)
config (objeto de configuração - opcional)
DELETE /api/marketplace/:id/uninstall Desinstala item do workspace id (UUID do item)
POST /api/marketplace/:id/rate Avalia um item (1-5 estrelas) rating (número 1-5)

Schema do Banco de Dados

Tabela: marketplace_items

Campo Tipo Descrição
id uuid Identificador único do item (PK)
name varchar(100) Nome do item
description text Descrição detalhada do item
author varchar(100) Autor ou organização criadora
category enum Categoria: agent, template, integration, plugin
iconUrl varchar(500) URL do ícone do item
bannerUrl varchar(500) URL do banner promotional
version varchar(20) Versão do item (ex: 1.0.0)
downloads integer Contador de downloads/instalações
rating decimal(3,2) Média de avaliação (0-5)
tags jsonb Array de tags para busca/filtro
configSchema jsonb Schema de configuração (campos necessários para instalar)
published boolean Se item está visível no marketplace
createdAt timestamp Data de criação
updatedAt timestamp Data de última atualização

Tabela: marketplace_installs

Campo Tipo Descrição
id uuid Identificador único da instalação (PK)
workspaceId uuid Workspace onde o item foi instalado (FK → workspaces)
itemId uuid Item instalado (FK → marketplace_items)
config jsonb Configurações específicas do workspace
installedAt timestamp Data da instalação
installedBy uuid Usuário que realizou a instalação (FK → users)

ConfigSchema - Exemplo

O campo configSchema define os campos de configuração necessários para instalar um item:

{
  "type": "object",
  "properties": {
    "apiKey": {
      "type": "string",
      "title": "API Key",
      "description": "Chave da API do serviço",
      "required": true
    },
    "webhookUrl": {
      "type": "string",
      "title": "Webhook URL",
      "description": "URL para receiving events",
      "required": false
    },
    "environment": {
      "type": "string",
      "enum": ["production", "staging", "development"],
      "default": "production"
    }
  }
}

Pontos de Risco

ALTO

ConfigSchema Malicioso

Um item pode definir configSchema solicitando permissões excessivas (ex: API keys, tokens de acesso, dados sensíveis).

Mitigação: Validar schemas antes da publicação, review manual, warn aos usuários sobre permissões solicitadas.
MÉDIO

Autores Falsos (Impersonation)

Indivíduos podem publicar itens se passando por autores conhecidos ou verified.

Mitigação: Sistema de verificação de autores, badge only para items auditados, report de abuse.
MÉDIO

Vulnerabilidades em Agents

Agents pré-configurados podem conter vulnerabilidades de segurança ou prompt injection.

Mitigação: Sandbox de execução, validação de outputs, audit de código dos items publicados.
MÉDIO

Isolamento de Workspace

Dados de config de um workspace podem vazar para outro se a lógica de filtro falhar.

Mitigação: Sempre filtrar por workspaceId, RLS (Row Level Security), audit de queries.
BAIXO

Rating Manipulado

Usuários podem manipular ratings criando múltiplas contas.

Mitigação: Uma avaliação por workspace, detecção de padrões suspeitos, média ponderada.
