Projeto Detalhe

Página de visualização detalhada de um projeto com abas para Overview, Agentes, Deploys, Código, Configuração, Segurança e Logs. Gerencia todas as informações e ações de um projeto específico.

O que é

O módulo Projeto Detalhe exibe informações completas de um projeto específico, incluindo métricas de performance, agentes associados, histórico de deploys, código fonte, configurações e análise de segurança. Permite gerenciamento completo do projeto através de abas interativas.

Objetivos

  • Exibir visão geral do projeto com métricas e atividade recente.
  • Listar e gerenciar agentes associados ao projeto.
  • Mostrar histórico de deploys com status e detalhes.
  • Permitir visualização de código fonte em tree view e editor.
  • Configurar parâmetros do projeto (framework, auto-deploy, etc).
  • Exibir análise de segurança e qualidade de código.
  • Acessar logs de execução do projeto.

Arquivos do módulo

O módulo está localizado em sistema-react/src/pages/ProjetoDetalhe/.

Arquivo Descrição
index.tsx Componente principal que gerencia tabs, carrega dados do projeto via slug e renderiza o header do projeto.
OverviewTab.tsx Tab de visão geral: timeline de atividades, métricas de performance (LCP, FID, CLS, TTFB), stack tecnológico e scores (Lighthouse, Coverage, Security).
AgentesTab.tsx Tab de agentes: lista cards de agentes com status (running/idle), última ação, tasks realizadas e timeline de atividades.
DeploysTab.tsx Tab de deploys: cards de estatísticas (total, último, uptime) e lista de deploys com status (success/failed/building).
CodigoTab.tsx Tab de código: file tree clicável e preview de código com numeração de linhas.
ConfigTab.tsx Tab de configurações: formulário para nome, framework, Node.js e toggles (auto-deploy, preview, auto-review, scan, docs).
SegurancaTab.tsx Tab de segurança: score de segurança, vulnerabilidades, dependências, checklists de segurança e qualidade.
LogsTab.tsx Tab de logs: visualização de logs de execução do projeto.
ProjetoDetalhe.module.css Estilos CSS modules do módulo.

Rota

A rota está definida em sistema-react/src/router.tsx.

  • GET /projetos/:slug - Página de detalhe do projeto onde :slug é o identificador único do projeto.

Exemplo de navegação:

<Link to="/projetos/meu-projeto-slug">Meu Projeto</Link>

Ou via useNavigate:

const navigate = useNavigate()
navigate(\`/projetos/\${projectSlug}\`)

Tipos de Dados (TypeScript)

Definidos em sistema-react/src/pages/ProjetoDetalhe/index.tsx.

Interface Agent

interface Agent {
  name: string        // Nome do agente (ex: "Codex", "QA Bot")
  emoji: string       // Emoji representando o agente
  status: 'running' | 'idle'  // Estado atual do agente
  tasks: number       // Número de tarefas realizadas
  lastAction: string  // Descrição da última ação
  time: string        // Tempo desde última ação
}

Interface Project

interface Project {
  name: string           // Nome do projeto
  icon: string           // Cor do ícone em hex
  emoji: string          // Emoji do projeto
  desc: string           // Descrição
  framework: string      // Framework (ex: "Next.js 14")
  branch: string         // Branch principal
  status: string         // Status (ex: "active")
  commits: number        // Total de commits
  branches: number       // Total de branches
  deploys: number        // Total de deploys
  uptime: string         // Porcentagem de uptime
  stack: string[]        // Stack tecnológico
  agents: Agent[]         // Agentes associados
  lcp: string            // Largest Contentful Paint
  fid: string           // First Input Delay
  cls: string            // Cumulative Layout Shift
  ttfb: string           // Time to First Byte
  lighthouse: number     // Score Lighthouse (0-100)
  coverage: number       // Cobertura de testes (%)
  securityScore: number  // Score de segurança (0-100)
  accessibilityScore: number  // Score de acessibilidade
  vulnerabilities: number // Número de vulnerabilidades
  dependencies: number   // Total de dependências
  outdated: number       // Dependências desatualizadas
}

Hooks React Query

O módulo utiliza useProjectBySlug de sistema-react/src/hooks/queries/useProjectQueries.ts para buscar dados do projeto.

import { useProjectBySlug } from '@/hooks/queries/useProjectQueries.ts'

const { data: project, isLoading } = useProjectBySlug(slug || '')

O hook retorna:

  • data - Dados do projeto vindos da API
  • isLoading - Estado de carregamento
  • error - Erro se falhar

Descrição das Abas

Overview (Visão Geral)

Exibe:

  • Agent Activity: Timeline com atividades recentes dos agentes (commits, reviews, deploys, etc)
  • Performance: Métricas Core Web Vitals (LCP, FID, CLS, TTFB) ou métricas de API
  • Stack: Tags com tecnologias do projeto
  • Scores: Barras de progresso para Lighthouse, Test Coverage, Security e Accessibility

Agentes

Exibe:

  • Agent Cards: Grid com cards de cada agente mostrando nome, emoji, status (running/idle), última ação, tasks e tempo
  • Agent History: Timeline com histórico de ações dos agentes (commits, reviews, testes, deploys, security scans, docs)

Deploys

Exibe:

  • Cards de stats: Total de deploys, último deploy (status), uptime
  • Lista de deploys: Tabela com ID, título, descrição, tempo e status (success/failed/building)

Código

Exibe:

  • File Tree: Árvore de arquivos clicável com ícones de pasta e tipo de arquivo
  • Code Preview: Visualizador de código com numeração de linhas e syntax highlighting

Arquivos mockados incluem: page.tsx, Hero.tsx, Header.tsx, Footer.tsx, api.ts, utils.ts, globals.css, package.json

Config (Configuração)

Exibe:

  • Formulário: Nome do projeto, Framework (dropdown), Node.js version
  • Toggles:
    • Auto-deploy (deploy automático ao fazer push)
    • Preview Deploys (deploys de preview para PRs)
    • Auto Code Review (revisão automática)
    • Continuous Scan (scan contínuo de segurança)
    • Auto Documentation (geração automática de docs)
  • Ações: Salvar alterações, Deletar projeto

Segurança

Exibe:

  • Cards de stats: Security score, vulnerabilidades, dependências
  • Security Analysis: Checklist de segurança (secrets, headers, HTTPS, rate limiting, CVEs, sanitização)
  • Code Quality: Checklist de qualidade (test coverage, code smells, strict mode, ESLint/Prettier, deps atualizadas, bundle otimizado)

Logs

Exibe logs de execução do projeto em formato de timeline.

Segurança e Acesso

  • O slug é usado para buscar o projeto no banco de dados filtrado por workspaceId.
  • O usuário deve ter acesso ao workspace que contém o projeto.
  • Caso o projeto não pertença ao workspace do usuário, retorna 404.
  • Variáveis de ambiente sensíveis devem ser expostas apenas quando necessário.

Pontos de Risco

ALTO

Alteração de configuração pode quebrar deploys

Modificar framework, branch ou variáveis de ambiente incorretamente pode causar falhas nos próximos deploys.

MÉDIO

Edição de código sem review

A visualização de código é apenas leitura, mas futuras implementações de edição direta podem introduzir erros sem revisão.

MÉDIO

Exposição de dependências desatualizadas

Dependências com vulnerabilidades conhecidas devem ser atualizadas rapidamente.

BAIXO

Delete acidental de projeto

O botão de delete deve ter confirmação para evitar exclusão acidental.

Como Usar

  1. Navegue até a lista de projetos em /projetos.
  2. Clique em um projeto para acessar /projetos/:slug.
  3. Na página de detalhe, explore as abas:
    • Overview: Veja métricas e atividades recentes.
    • Agentes: Monitore agentes e suas atividades.
    • Deploys: Acompanhe o histórico de deploys.
    • Código: Navegue pela estrutura de arquivos.
    • Config: Ajuste configurações do projeto.
    • Segurança: Verifique scores e vulnerabilidades.
    • Logs: Acesse logs de execução.
  4. Na aba Config, você pode:
    • Editar nome e framework do projeto
    • Ligar/desligar auto-deploy
    • Ativar preview deploys
    • Configurar scan contínuo de segurança
    • Salvar ou deletar o projeto
